Claude /usage migration: UI states and copy

PromptJuice replaces the Claude status-line bridge with Claude Code's built-in /usage command. This is the final adjudicated spec: every user-visible string, state, action, and gate, reconciled across four review rounds with the implementing agent.

01 · Recommendation

What ships and why

  • Model: three independent axes replace the bridge-era enum: ClaudeAccessState (checking, cliMissing, updateRequired, signedOut(reason: initial | reauthenticationRequired), subscription, apiBilling, externalProvider, unsupportedAuth, authCheckFailed) × ClaudeRefreshState (idle, refreshing, backingOff(nextAttemptAt), failed) × the snapshot (source, confidence, updatedAt, windows). LegacyBridgeStatus is a fourth, independent migration axis.
  • One freshness vocabulary: Updated just now → Updated 12 min ago → Updated at 3:14 PM → Updated yesterday at 3:14 PM → Updated Jul 18 at 3:14 PM. Exact, stale, cached, and earlier never render; they survive internally for cache logic.
  • Backoff is client-owned: the coordinator's 5→15→30→60-minute ladder always yields a valid nextAttemptAt, so copy is always next check at 3:45 PM and no unknown-retry state exists. "Rate limited" appears nowhere on rows.
  • 2.1.208 cached bars are a saved reading: when the usage endpoint is rate-limited, Claude Code renders last-known bars with an "as of" time. That parses as a saved snapshot with the real measurement timestamp, never as a failed probe.
  • Three action journeys, journey-labeled everywhere: Install / Sign in / Update. "Set up" survives only as an umbrella concept, never as an action label.
  • Five auth categories: subscription · first-party Console/API · external provider (Bedrock, Vertex, Foundry, gateway) · signed out · unknown. /usage runs only for confirmed subscription. Neutral categories get visible, severity-excluded rows.
  • Fresh window retired for Claude (no signal exists under /usage); Codex keeps its app-server fresh-window signal, provider-gated so Claude data can never trigger the copy.
  • Advanced pickers deferred out of v1. Smart cadence runs automatically. A future cadence control would be labeled "Refresh Claude usage".
  • One release gate: the privacy copy is target copy gated on an estimator refactor (G1–G4). The Anthropic clarification (B5) is deferred as an optional courtesy — not a gate — with the passive-data fallback ready.

How this number is measured

PromptJuice reads your Claude plan usage with Claude Code's built-in /usage command. These are the same numbers Claude shows you. Claude Desktop, Claude.ai, and Claude Code share one plan allowance, so this covers all of them.

PromptJuice asks Claude Code for your plan usage about every 15 minutes and sends no model prompt. When an estimate is needed, it scans local Claude Code activity records and extracts usage totals and timestamps. It does not store, display, or transmit conversation text.

Journey-scoped rechecks (Check Again and the ≥30s-debounced app-activation re-probe; neither ever invokes /usage — usage refresh belongs to the coordinator): CLI missing → locator + version + auth · Update required → version + auth · Signed out → auth only. Install → Sign in transition: when the open Install sheet's recheck finds the CLI present but signed out, the sheet advances to the Sign In journey in place — a two-stage flow, never a dead end. Copy and Open Terminal copies the command and launches macOS's built-in Terminal.app (com.apple.Terminal by bundle id; may activate an existing window — copy promises only "opens Terminal"); the command is also on the clipboard for iTerm/Warp users. Long commands sit in a contained block that wraps inside the sheet; Check Again and every button stay one line; the sheet footer also gets native SwiftUI snapshots at minimum macOS text size and with accessibility text enlargement. Commands verified against the official docs (setup, cli-reference): install curl -fsSL https://claude.ai/install.sh | bash; sign in claude auth login (documented shell sign-in; flags --email/--sso/--console; /login is the in-TUI alternative); update chosen by executable provenance — native claude update, Homebrew brew upgrade claude-code, npm npm install -g @anthropic-ai/claude-code@latest, unknown provenance → the sheet shows the resolved executable path plus native, Homebrew, and npm alternatives as three separate full-width command blocks each with its own copy affordance, and the primary action becomes Open Terminal, copying nothing automatically (Copy and Open Terminal is reserved for a known provenance with one selected command); never copy a guess. Reauthentication (expired or revoked OAuth) reuses the Sign In sheet unchanged with the subtitle Signed out of Claude Code · sign in again. Advanced pickers remain deferred from v1 (annotation only — the note no longer appears inside the Settings window mockup). No live polling or auto-flipping footer in v1.

Aggregate rules

Only quota-bearing snapshots (subscription readings including estimates, and Codex) feed aggregate severity, menu-bar percentage, glyph, and notifications. Neutral rows are enabled, visible, non-quota-bearing. With no quota-bearing snapshot anywhere: muted droplet, no percentage badge, header Claude plan usage unavailable with the category detail — never healthy, never 100%. With Codex connected, Codex alone drives everything aggregate.

06 · Gates

One release gate, the optional B5 note, and the auth rule

Privacy-copy gate (G1–G4)

The popover privacy paragraph is TARGET copy. The shipping estimator uses JSONSerialization over bounded JSONL records with nested dictionary access. The copy ships only when: G1 the estimator is refactored to narrow typed decoding (or an equivalent extraction boundary) that immediately discards conversation-bearing fields, retaining only timestamps, request/message identifiers for deduplication, sidechain state, and token totals; G2 raw JSONL records and conversation fields can never enter logs, caches, errors, fixtures, analytics, or UI state; G3 tests prove persisted snapshots contain only derived usage information; G4 a repository-wide logging review passes.

Optional B5 compliance note (deferred, not a gate)

Product-risk analysis, not legal advice: PromptJuice contains no Claude HTTP client and receives no Claude credential; it launches the user-installed Claude Code locally; Claude Code may make its own authenticated request to Anthropic's usage endpoint when /usage runs; PromptJuice parses the resulting terminal screen; the estimator performs local file reads and sends no log content anywhere. Comparable Claude-usage tools ship without formal clarification, and this approach is more conservative than most (no credentials, no model calls), so a hard gate is stricter than the ecosystem norm. Decision: the clarification is an OPTIONAL courtesy, not a blocker. Local, dogfood, and limited use proceed freely. For a broad public launch, optionally send the note, or ship with the passive-data fallback (estimator-only, no PromptJuice-initiated /usage) ready as a backstop. Not being pursued now.

Credential policy D (strengthened — B1 approved)

No environment mutation. Classification runs against the exact child process PromptJuice launches, enumerating the known documented billing-affecting sources for supported Claude Code versions (future releases may add sources; unknown combinations fail closed): app-visible env (ANTHROPIC_API_KEY, ANTHROPIC_AUTH_TOKEN, CLAUDE_CODE_OAUTH_TOKEN, ANTHROPIC_BASE_URL, provider flags, gateway/proxy vars), settings.json (apiKeyHelper, env block, provider/base-URL settings), managed settings, and claude auth status as corroborating evidence, never sole proof. Subscription requires auth status agreement AND no overriding credential source. Forward-compatible rule: additive unrelated JSON fields are ignored; unknown values of billing-relevant discriminators (authMethod, apiProvider, subscriptionType) and missing required billing evidence fail closed to unknown/unsupported, which skips /usage. No separate /status probe in v1; in-band plan labels on the /usage screen serve as confirmation when present.

07 · Conventions and accessibility

Copy rules

  • Freshness: the five-tier Updated formatter, relative under an hour. Backoff: next check at {time}. Failure: having trouble {updating | checking}. Estimate keeps ~ and the word Estimate.
  • Rows say what; tooltips add when and why in one line with ·; the popover gets full sentences; footnotes are the only multi-line explanations outside it.
  • "Claude Code" is the CLI, "Claude" the account. No bridge, statusline, PTY, or terminal-dependency language; /usage only in the popover body. Buttons Title Case with spelled-out "and"; prose and phrase-like menu items sentence case; no em dashes; SF Symbols only (info.circle, clock, doc.on.doc).
  • The withdrawn claims, banned everywhere: Nothing is sent to Claude · Your Claude quota is fine. · never reads your conversation text · PromptJuice itself makes no network calls.

08 · Fixtures and sequence

Fixture plan and implementation order

Category 1, synthetic (authorable now): flat /usage screens (session only, session+weekly, 0% used, ANSI residue, truncated, malformed), cached-bars set F-RL1–F-RL4 plus "as of" timestamp variants (12-hour, 24-hour locale, timezone/DST transition, ANSI residue inside the timestamp, malformed), auth status permutations (additive harmless fields, missing required fields, unknown discriminator values, expired/revoked-OAuth reauth shape), version strings, locator/provenance outcomes (native, Homebrew Apple Silicon, Homebrew Intel, npm, custom symlink, unknown), corrupt persisted nextAttemptAt. Probe results can carry BOTH a usable snapshot and the backoff signal. Category 2, sanitized OSS: CodexBar fixtures with provenance and license recorded. Category 3, live: Codex prepares the capture harness but never executes it and never invokes Claude Code; Jeremy runs the approved read-only captures (subscription /usage, auth status, version) after reviewing the script. The harness uses an owner-only (0700) scratch directory containing a dedicated empty working directory — every Claude Code capture process runs from that controlled cwd — with owner-readable (0600) raw files; redacts emails, org/account IDs, UUIDs, usernames, and absolute paths; and preserves the lexical timestamp format, timezone shape, ANSI structure, and surrounding parser-relevant text — it never normalizes away the syntax a fixture is meant to prove. Jeremy inspects sanitized output before anything enters the repository. Console login changes and genuine rate-limit captures remain separately approved future work; the auth classifier and its expected field names (loggedIn, authMethod, apiProvider, subscriptionType) stay provisional until Jeremy's capture confirms them.

Sequence (binding — B1/B2/B3 approved, B5 deferred · design ready, execution awaits Jeremy's explicit start instruction)

0 consolidate this spec into the worktree + banner claude-usage-plan.md → 1 synthetic fixtures, OSS evaluation, capture harness PREPARED, then a hard stop: Codex reports the fixture inventory, CodexBar license verdict, harness location, and Jeremy's review/run instructions, and Jeremy decides when to run the captures (Codex never executes the harness) → 2 locator, version gate, provenance detection, fail-closed auth classification (provisional until the capture) → 3 flat PTY transport with fixed command allowlist + pure /usage parser → 4 access/refresh state, client-owned cooldown, cache, coordinator, source ladder behind the dogfood switch (bridge stays hidden fallback) → 5 G1–G4 estimator privacy refactor → 6 presentation layer per this spec, incl. journey-scoped rechecks, the Install→Sign in transition, and native snapshots of the sheet footer at minimum text size and with accessibility enlargement → 7 dogfood; remove Jeremy's verified bridge via the Remove path → 8 delete the fallback bridge subsystem + repo-wide string sweep → 9 optional, unscheduled: the B5 note if a broad public launch approaches.

09 · Decisions

Resolved

  • B1 — APPROVED. Credential policy D as strengthened.
  • B2 — APPROVED. Minimum Claude Code 2.1.208 and the update-journey behavior.
  • B3 — APPROVED (read-only captures). Capture and commit sanitized real output for the parser/classifier tests; each capture approved individually, login-state captures need a written isolation and recovery plan first.
  • B5 — DEFERRED / OPTIONAL. The Anthropic clarification is a courtesy, not a blocker: comparable tools ship without it and PromptJuice is more conservative (no credentials, no model calls). Ship dogfood and limited use freely. For a broad public launch, optionally send the clarification, or ship with the passive-data (estimate-only) fallback ready. Not doing now.
  • Bridge — RESOLVED. Never shipped externally; needs-review path dropped, ownership-verified Remove ships for the author's own machine.
  • Ampersand — RESOLVED. Keep "Copy and Open Terminal" (Apple Style Guide spells out "and" for action buttons; "&" is for two-noun category names).
